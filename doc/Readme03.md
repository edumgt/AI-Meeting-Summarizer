# Readme03 - AI 모델 스택(Hugging Face Transformers)

## 1. 모델 계층 구성
이 프로젝트는 작업별 모델 분리 전략을 사용합니다.

- 요약 모델(`SUM_MODEL_ID`)
- 분류 모델(`CLS_MODEL_ID`)
- 임베딩 모델(`EMB_MODEL_ID`)

작업별로 최적화된 모델을 분리하면 품질/속도/유지보수 측면에서 유리합니다.

## 2. 요약 모델
- 기본 예시: `gogamza/kobart-summarization`
- 한국어 문서 요약에 특화
- 긴 문서 입력 시 토큰 제한이 있으므로 청크 기반 전처리가 필수

## 3. 분류 모델
- 기본 예시: `Seonghaa/korean-emotion-classifier-roberta`
- 감정/분위기 등 회의 맥락 분석에 활용 가능
- 회의 참여자 발화 품질 모니터링 지표로 확장 가능

## 4. 임베딩 모델
- 기본 예시: `upskyy/bge-m3-korean`
- 문장/문서를 벡터로 변환하여 의미 기반 검색 가능
- 향후 회의록 검색, 유사 케이스 추천 기능의 기반

## 5. 모델 운영 전략
- 환경변수 기반 모델 스위칭으로 배포 유연성 확보
- 초기 로딩 시간(warm-up) 고려 필요
- CPU 실행 시 추론 시간 증가를 감안한 API 타임아웃 설계 필요

## 6. 품질 개선 아이디어
- 도메인 데이터(사내 회의록)로 파인튜닝
- 요약 품질 측정(ROUGE + 휴먼 평가)
- 분류 임계값 조정 및 다중 라벨 구조 검토
