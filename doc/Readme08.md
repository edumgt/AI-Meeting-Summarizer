# Readme08 - 설정/환경변수/의존성 관리

## 1. 환경변수 전략
모델 및 파이프라인 관련 핵심 파라미터를 코드 하드코딩이 아니라 환경변수로 외부화했습니다.

- `SUM_MODEL_ID`
- `CLS_MODEL_ID`
- `EMB_MODEL_ID`
- `CHUNK_MAX_TOKENS`
- `REDUCE_MAX_TOKENS`

## 2. 장점
- 배포 환경별 설정 분리 용이
- 모델 실험(A/B 테스트) 속도 향상
- 코드 변경 없이 운영 파라미터 조정 가능

## 3. 의존성 버전 관리
- Python: `requirements.txt`로 버전 범위 또는 고정
- Node.js: `package.json`, `package-lock.json`으로 잠금
- 라이브러리 호환 이슈 시 명시적 다운그레이드 전략 적용 가능

## 4. 권장 운영 정책
- 운영 환경은 “고정 버전” 원칙 사용
- 개발 환경만 제한적으로 유연 버전 허용
- 변경 이력은 CHANGELOG 또는 릴리즈 노트로 추적

## 5. 보안 관점
- `.env` 파일은 Git 추적 제외
- 민감정보(키/토큰) 주입은 시크릿 매니저 사용 권장
- 기본값과 운영값을 명확히 분리
