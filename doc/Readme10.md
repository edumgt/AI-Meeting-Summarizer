# Readme10 - 확장 로드맵 및 실무 적용 시나리오

## 1. 단기 로드맵(1~2개월)
- API 응답 표준화 및 에러 코드 정비
- 간단한 웹 UI에서 파일 업로드/결과 보기 지원
- 요약 결과 템플릿(회의 목적/결정사항/액션아이템) 고도화

## 2. 중기 로드맵(3~6개월)
- 벡터 DB 연동으로 회의록 검색/질의응답 구축
- 사용자/조직 단위 권한 관리 도입
- 배치 파이프라인(하루 회의록 자동 요약) 스케줄링

## 3. 장기 로드맵(6개월~)
- 사내 데이터 기반 도메인 특화 파인튜닝
- 음성(STT) → 요약 자동화 파이프라인 통합
- 대시보드 기반 회의 생산성 분석(팀별/기간별)

## 4. 실무 적용 패턴

### 패턴 A: 팀 단위 회의 자동 정리
- 회의 종료 후 텍스트 업로드
- 3분 내 요약/액션아이템 공유
- 누락 과제 방지 및 결정 이력 축적

### 패턴 B: 기술 조직 지식 검색
- 임베딩 저장 후 유사 회의 검색
- 과거 의사결정 재활용으로 중복 논의 감소

### 패턴 C: 운영 리스크 모니터링
- 분류 결과를 기반으로 부정/리스크 신호 감지
- 이슈 확산 전 조기 대응

## 5. 결론
AI-Meeting-Summarizer의 기술 스택은 “빠른 도입 + 점진적 확장”에 최적화되어 있습니다. 초기에는 로컬 CPU 기반 경량 운영으로 시작하고, 필요 시 컨테이너/벡터DB/모델 고도화 방향으로 자연스럽게 성장시킬 수 있습니다.
