# Readme09 - 품질관리: 테스트, 관측성, 성능 최적화

## 1. 테스트 전략

### 1-1. 기능 테스트
- `/health` 상태 확인
- 샘플 스크립트 기반 요약/분류/임베딩 호출 검증
- `/report` 결과 markdown 생성 검증

### 1-2. 회귀 테스트
- 모델 변경 전후 동일 입력 비교
- 요약 길이, 키포인트 보존 여부, 분류 스코어 편차 확인

## 2. 관측성(Observability)
- 요청 단위 처리 시간 로깅
- 에러 유형(Validation/Model/Timeout) 분리
- 모델 로딩 시간 및 메모리 사용량 모니터링

## 3. CPU 환경 성능 최적화
- 입력 전처리(불필요 공백/노이즈 제거)
- 청크 크기 튜닝으로 호출 횟수와 문맥 품질 균형화
- 배치 처리 가능 시 micro-batching 검토

## 4. 품질 지표 예시
- 요약: ROUGE, 핵심 의사결정 문장 포함률
- 분류: 정확도/F1
- 임베딩: 검색 정밀도(Top-k hit rate)

## 5. 운영 안정화 팁
- 스타트업 시 model warm-up 요청 1회 실행
- 대형 문서 요청에 대한 timeout/재시도 정책 수립
- 실패 케이스 샘플을 축적해 지속 개선
